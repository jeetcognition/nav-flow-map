# Engineering Audit — nav-flow-map

Date: 2026-07-17 · Branch: `newui` · Phase 0 (read-only audit, no code changed)

This repo contains **two apps plus a serverless worker**:

| Part                                                      | Stack                                         | State                                             |
| --------------------------------------------------------- | --------------------------------------------- | ------------------------------------------------- |
| Root legacy app (`index.html`, `bugs.js`, `testcases.js`) | Vanilla JS, no build, Cytoscape via unpkg CDN | Live on GitHub Pages                              |
| `worker/`                                                 | Cloudflare Worker (`worker.js`, 124 lines)    | Deployed at `navmap-save.jeet-navmap.workers.dev` |
| `app/` "QA Command Center"                                | React 19 + Vite 8 + TS 6, oxlint              | Phase 1, UI + mock fixtures only                  |

---

## 1. What the app does

### Root legacy app

Interactive nav-flow tree of an enterprise web app, each page node linked to QA test cases and bugs.

- **Entry point:** `index.html` — ~1,390 lines: inline CSS + ~1,000 lines of inline JS. `testcases.js` (~109 KB, single-line array of ~220 cases) and `bugs.js` load via `<script>` tags; Cytoscape 3.30.2 loads from unpkg (pinned version, but internet-required, no SRI hash).
- **Data flow:** `BASE_PAGES` (49 pages hard-coded in `index.html:405-449`) + committed `navmap-edits.json` (fetched at startup, `index.html:1380-1388`) + localStorage overlay (`navmap-edits-v1`). "Save to repo" POSTs merged edits to the worker, which commits `navmap-edits.json` via the GitHub API and, if the save contains promotable edits, starts a Devin session that rewrites drafts into the canonical sources (`testcases.js`, `bugs.js`, `index.html`, `qa-testing/testcases/*.md`) and commits **directly to `main`**.
- **localStorage keys (5):** `navmap-edits-v1`, `navmap-expanded-branches-v1`, `navmap-theme`, `navmap-sidebar`, `navmap-panel-width`.

### `app/` QA Command Center

React SPA: Dashboard, Bugs (table + kanban), Incidents, Runs, Automation, GraphMap, Settings — 10 routes (`App.tsx:20-43`).

- **Data layer:** `src/data/dataService.ts` — singleton in-memory store over typed JSON fixtures (`src/data/fixtures/`, generated by `scripts/seed.mjs`), pub/sub via `useDataVersion()`. UI never imports fixtures directly — clean swap point for a real API.
- **AI layer:** `src/data/aiService.ts` — `AI_MOCK = true` (line 7); every function returns canned results behind an 800–1300 ms fake delay. Designed to be replaced with real Anthropic API calls in Phase 2.

## 2. Findings

### 2.1 Secrets — ✅ clean

- No committed tokens anywhere (scanned for `ghp_`, `github_pat_`, `sk-`, AWS keys, bearer literals).
- Worker reads `env.GITHUB_TOKEN` (`worker.js:101`) and `env.DEVIN_API_KEY` (`worker.js:54`) from Cloudflare secret bindings; `wrangler.toml` contains no secrets.
- ⚠️ Not secret but sensitive-adjacent: Devin **org ID hard-coded in the API URL** (`worker.js:51`).

### 2.2 Hard-coded values & magic numbers

| Location                                     | Value                                                                                                       | Should be                             |
| -------------------------------------------- | ----------------------------------------------------------------------------------------------------------- | ------------------------------------- |
| `index.html:1250`                            | Worker endpoint URL                                                                                         | config constant / build-time env      |
| `worker.js:1-4`                              | repo, filename, CORS origins, 512 KB limit                                                                  | `wrangler.toml` vars                  |
| `worker.js:51`                               | Devin API URL incl. org ID                                                                                  | env var                               |
| `index.html:379,770,772,906-908,1149`        | layout magic numbers (layer depth 4, gaps 26/52/80, fit padding 0.14/40/120, panel clamp 260/-200)          | named constants                       |
| `app/` (15 sites)                            | `"enterprise"` surface ID scattered across Automation/Bugs/Dashboard/GraphMap/NodePanel/CreateTestcaseModal | `DEFAULT_SURFACE` constant or context |
| `app/dataService.ts:109`, `aiService.ts:122` | coverage threshold 0.6, risk threshold 3                                                                    | named constants                       |

### 2.3 Error handling

**Legacy app — silent failures:**

- `index.html:385-389, 455-456` — corrupted localStorage silently resets branch state / **discards all local edits** with no warning.
- `index.html:1382` — failed fetch of `navmap-edits.json` indistinguishable from "file absent" (`.catch(() => null)`).
- `index.html:1261` — malformed save response swallowed (`.catch(() => ({}))`).
- 12 `alert()` calls for all validation and save errors — modal-blocking, no retry, no logging.

**Worker:**

- Shallow validation only: required keys + size + JSON parse (`worker.js:88-97`). No semantic validation (page IDs, link cycles) — a buggy client can commit corrupt state that a Devin session then "promotes".
- **No rate limiting** — any allowed origin can spam GitHub commits and paid Devin sessions.

**React app:**

- Missing `.catch()` on AI promises: `Bugs.tsx:394-398` (duplicate scan), `CreateTestcaseModal.tsx:38-48` (draft), `RunDetail.tsx:68-74` (summary) — on rejection the UI hangs in loading state forever. Harmless while `AI_MOCK = true`; **a Phase 2 blocker** the moment real API calls land.
- No error UI states anywhere (`CoverageGapCard`, `CandidatesCard`, modals).

### 2.5 Duplication

- **Legacy:** `alert(...); return;` validation ×8; `saveEdits(); buildGraph(); showPage(...)` ×4; `allPages().find(x => x.id === …)` ×9; pages-by-id map rebuilt ×3.
- **React app:** filter-bar pattern reimplemented 3× (`Automation.tsx:256-292`, `Runs.tsx:103-147`, `Incidents.tsx:247-305`); `badges.tsx:19-25` duplicates `lib/bugStatus.ts` status→label/color mapping; AI-draft-modal skeleton/edit/save flow duplicated between `CreateTestcaseModal` and the inline `DraftBugModal` in `Bugs.tsx`.

### 2.6 Dead code — minor

- Codebases are surprisingly clean: no unused deps (`framer-motion`, `recharts`, `dagre`, `@xyflow/react` all used), no unused components, no `@ts-ignore`, no `any`.
- Minor: `reach` field in `testcases.js` rendered but load-bearing nowhere else; `navmap-edits.json` promotable keys currently all empty (by design — drained after promotion).

### 2.7 Types, lint, format

- `app/tsconfig.app.json` has **no `strict` mode** — no `strictNullChecks`, no `noImplicitAny`. 29 `as` casts (mostly justified fixture-JSON casts), 1 non-null assertion (`main.tsx:10`).
- `.oxlintrc.json` enables exactly **2 rules** (`rules-of-hooks`, `only-export-components`).
- No formatter (no Prettier config); no lint/format/type tooling at all for the root app and worker (plain untooled JS).

### 2.8 Tests & CI — none

- Zero test files in the entire repo. No test runner. No `.github/` directory, no CI of any kind. `testcases.js` could be broken by a bad commit (including by the automated Devin promotion, which commits straight to `main`) and nothing would catch it before the live site loads it.

### 2.9 Dependencies

- `app/package-lock.json` exists ✓; deps use caret ranges (`^`) — acceptable with lockfile, but no audit gate.
- Root app has **no package.json at all** — Cytoscape comes from unpkg at runtime (no SRI integrity hash, availability risk).
- No Dependabot/Renovate, no `npm audit` in any pipeline.
- Vite build emits a single 1.4 MB JS bundle (390 KB gzip) — over Vite's 500 KB chunk warning; graph/chart libs are candidates for dynamic import.

### 2.10 Process risks (noteworthy)

- The Devin auto-promotion pipeline commits **directly to `main`** with no CI, no review, no branch protection. One bad AI commit takes down the live site.
- `main` has no branch protection (direct pushes possible).

## 3. Proposed refactor plan (behavior-preserving)

Each phase = small, single-concern PRs. Nothing merges red once CI exists.

### Phase 1 — Safety net

1. **Repo tooling baseline**: root `package.json` (private workspace-style), Prettier config repo-wide; ESLint for root `index.html` JS + `worker/`; extend `.oxlintrc.json` (correctness + import rules) for `app/`.
2. **TypeScript strict**: enable `strict: true` in `app/tsconfig.app.json`; fix fallout (est. small — code is already cast-light).
3. **Characterization tests**:
   - `app/`: Vitest + React Testing Library — golden-path tests for Dashboard render, Bugs filter/table/board, dataService mutations, aiService mock outputs.
   - Legacy: extract pure functions (`mergedEdits`, `KIND_MAP` mapping, edit-merge logic) is Phase 2; for now add Node-based tests that `node --check` `testcases.js`/`bugs.js`, validate `navmap-edits.json` schema, and validate every case-ID prefix maps to a `BASE_PAGES` prefix (this is exactly what the Devin pipeline can break).
   - `worker/`: Vitest + `unstable_dev`/miniflare tests for CORS, method, validation, size-limit paths.
4. **CI (GitHub Actions)**: lint + typecheck + test + `app` build on every PR and on push to `main` (catches the direct Devin commits after the fact, at minimum).

### Phase 2 — Structure & correctness

5. **Config extraction**: worker constants → `wrangler.toml` vars + env (Devin org/URL, repo, origins); legacy app endpoint → single config block; `app/` `DEFAULT_SURFACE` + thresholds → `lib/constants.ts`.
6. **Split `index.html`**: extract inline JS to modules (`data.js`, `graph/layout.js`, `graph/routing.js`, `panel.js`, `edits.js`, `main.js`) and CSS to a stylesheet — still no build step required (plain ES modules). Break up `buildGraph`/`layoutTree`/`showPage`.
7. **Split `app/` god pages**: Automation → 5 components, Bugs → 4, Incidents → 2; shared `<FilterBar>`; consolidate badge/status maps onto `lib/bugStatus.ts`.
8. **Error handling**: `.catch()` + error states on all AI promises; replace `alert()`s with a small toast helper; warn on localStorage-corruption recovery; add semantic validation + rate limiting (or Turnstile) to the worker. _(Some of these are behavior changes — flagged as separate, clearly-labeled PRs.)_

### Phase 3 — Operability & docs

9. Root README rewrite covering both apps + worker deploy; `CONTRIBUTING.md`; pre-commit hooks (lint-staged + gitleaks); `TODO.md` backlog (bundle splitting, worker rate limiting, SRI for CDN, localStorage persistence of surface selection).

### Phase 4 — Enforcement

10. Branch protection on `main` (PRs + green CI required) — **note: this conflicts with the Devin auto-promotion pipeline, which commits directly to `main`. Decision needed:** route promotions through a bot PR with auto-merge-on-green, or exempt the bot.
11. Coverage ratchet in CI; PR template; `AGENTS.md` conventions; CODEOWNERS; Dependabot; secret scanning (gitleaks) in CI.

### Known bugs found (would be separate labeled PRs, not silent fixes)

- AI promise rejections leave `app/` UI in permanent loading state (latent until Phase 2 API work).
- Legacy save flow can't distinguish network failure from missing edits file; failed saves lose no data but give a truncated, unactionable error.

---

## 4. Decisions (made 2026-07-17)

1. **Devin pipeline vs branch protection**: promotions will flow through a **bot PR with auto-merge on green CI** (worker prompt updated in Phase 4); branch protection stays absolute.
2. **Legacy app scope**: **safety net only** — tests, schema validation, config extraction, error-handling fixes. The full modular split of `index.html` (Phase 2, item 6) is dropped; `app/` is the successor.
3. Phase 1 approved and started 2026-07-17.
